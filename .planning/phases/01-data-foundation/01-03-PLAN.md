## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| London Datastore → Application | CSV data from external source |
| Geocoding API → Application | External API for postcode to lat/lon conversion |
| Import Script → Database | Bulk data insertion |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Spoofing | London Datastore CSVs | mitigate | Validate CSV structure and data types before import |
| T-01-03-02 | Tampering | Geocoding API | accept | Using free public API (Nominatim), data is public anyway |
| T-01-03-03 | Repudiation | Import Logs | mitigate | Log all import operations (inserted, duplicates, errors) |
| T-01-03-04 | Information Disclosure | Venue Data | accept | Venue data is public information |
| T-01-03-05 | Denial of Service | Geocoding API | mitigate | Cache geocoding results, rate limit API calls, handle failures gracefully |
| T-01-03-06 | Elevation of Privilege | Database Insert | mitigate | Use parameterized queries, validate all data before insert |

**Critical Mitigations for This Plan:**
- Validate CSV structure and data types before import
- Use parameterized queries for all database inserts
- Cache geocoding results to avoid API abuse
- Log all import operations for auditability
- Handle geocoding failures gracefully (skip venue, log error)
</threat_model>
