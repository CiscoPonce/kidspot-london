<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| VPS → External | VPS exposes ports 4000 (API), 5432 (Postgres), 6379 (Redis) to internet |
| Docker → Host | Container filesystem access via volumes |
| Environment → Containers | Secrets injected via environment variables |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | API Service | mitigate | Implement API authentication in Phase 3 (API-03) |
| T-01-02 | Tampering | Docker Images | mitigate | Use official images (node:20-slim, postgis/postgis, redis:alpine) with specific tags |
| T-01-03 | Repudiation | API Service | mitigate | Add request logging in Phase 3 (API-03) |
| T-01-04 | Information Disclosure | PostgreSQL Port | mitigate | Bind postgres port only to docker network (remove ports mapping or bind to 127.0.0.1) |
| T-01-05 | Denial of Service | Redis | accept | Redis has no auth in dev, acceptable for MVP. Add AUTH in production if needed. |
| T-01-06 | Elevation of Privilege | Docker Containers | mitigate | Run containers as non-root user (add USER node in Dockerfiles) |

**Critical Mitigations for This Plan:**
- Remove PostgreSQL port mapping from docker-compose.yml (bind only to docker network)
- Add non-root user to both Dockerfiles
- Document security requirements in README.md
</threat_model>
