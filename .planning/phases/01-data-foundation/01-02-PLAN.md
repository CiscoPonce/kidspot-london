  <files>backend/db/schema.sql</files>
  <action>Create backend/db/schema.sql with:
1. Enable PostGIS extension: CREATE EXTENSION IF NOT EXISTS postgis
2. Enable fuzzystrmatch extension: CREATE EXTENSION IF NOT EXISTS fuzzystrmatch
3. Create venues table with columns:
   - id BIGSERIAL PRIMARY KEY
   - source TEXT NOT NULL (datastore, osm, scraper_smart, brave_fallback)
   - source_id TEXT UNIQUE
   - name TEXT NOT NULL
   - type TEXT NOT NULL (softplay, community_hall, park, other)
   - address TEXT
   - postcode TEXT
   - phone TEXT
   - website TEXT
   - description TEXT
   - lat DOUBLE PRECISION NOT NULL
   - lon DOUBLE PRECISION NOT NULL
   - location GEOGRAPHY(POINT,4326) GENERATED ALWAYS AS (ST_MakePoint(lon, lat)) STORED
   - is_verified BOOLEAN DEFAULT TRUE
   - last_fetched TIMESTAMPTZ DEFAULT NOW()
   - created_at TIMESTAMPTZ DEFAULT NOW()
   - updated_at TIMESTAMPTZ DEFAULT NOW()
4. Create spatial index: CREATE INDEX idx_venues_location ON venues USING GIST(location)
5. Create type index: CREATE INDEX idx_venues_type ON venues(type)
6. Create is_duplicate_venue function (from technical_overview.md section 2.1)

Use the exact schema from technical_overview.md section 2.1.</action>
