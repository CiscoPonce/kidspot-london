# Phase 07.5-01 - TypeScript Migration for Worker

## 1. Objective
Convert `worker.js` to `worker.ts`, update `Dockerfile.worker` to use `tsx` or a compilation step, and ensure environment variables (`YELP_API_KEY`, etc.) are mapped and validated.

## 2. Requirements
- **TS-01**: Convert `worker.js` to `worker.ts` and ensure all environment variables are correctly mapped.

## 3. Tasks
1. **Convert Worker Code**: Rename `backend/src/worker.js` to `backend/src/worker.ts`. Add explicit types for BullMQ job data and return values.
2. **Update Ecosystem & Docker**: Update `backend/ecosystem.config.js` and `backend/Dockerfile.worker` to run the `.ts` file via `tsx` or compile it first.
3. **Environment Variables**: Audit `backend/src/config/env.ts` (or similar config) to ensure `YELP_API_KEY`, `DB_PASSWORD`, and other required variables are explicitly typed and checked at startup.

## 4. Dependencies
- None.

## 5. Verification
- `docker-compose up worker --build` starts successfully.
- No TypeScript compilation errors (`npx tsc --noEmit`).
- Worker successfully connects to Redis and PostgreSQL.
