Project Overview – “KidSpot London” (v3 - Comprehensive)1. Executive Summary“KidSpot London” is a hyper-local, community-driven search engine designed to solve the data fragmentation problem surrounding child-friendly venues in London. By combining public datasets, intelligent LLM-powered scraping, and real-time web search fallbacks, KidSpot provides parents and event organizers with an instant, geo-aware directory of safe, age-appropriate locations for parties and gatherings.2. Deep Problem StatementFinding a venue for a child’s birthday party in London is a highly inefficient process.Data Fragmentation: Information is scattered across local council websites (leisure centres), independent business pages (soft-play), and parish sites (church halls).Data Decay: Operating hours and pricing change frequently, and manual directories become stale within months.The "Zero-Result" Frustration: Existing directories often yield zero results if the user searches a specific postcode with a tight radius, forcing them back to generic web searches.Lack of Geo-Context: Parents need to know exactly how far a venue is, what the parking situation is like, and what amenities are nearby.3. Vision & MissionVision: To become the default, zero-friction utility for parents in the UK to discover, evaluate, and share child-friendly spaces, starting with London.Mission: 1. Unify the Data: Aggregate public datasets (London Datastore), open-source mapping (OSM), and unstructured web data into a single, highly structured PostgreSQL database.2. Guarantee a Result: Ensure no user ever hits a "dead end" by integrating live API fallbacks.3. Keep it Free & Accessible: Maintain a mobile-first, lightning-fast UI that requires no account creation or paywalls for core usage.4. Comprehensive User Stories (MVP & Beyond)Search & DiscoveryUS-01: As a parent, I want to type my street name, postcode, or click "Use My Location" to set my search epicenter.US-02: As a parent, I want to adjust a slider (1 to 10 miles) so I can control how far I am willing to travel.US-03: As a user, if the internal database has no venues within my radius, I want the system to automatically fetch and display nearby web results so I still get useful leads.US-04: As an organizer, I want to filter the results by category (e.g., "Soft Play", "Community Centre", "Outdoor Park") to find the right vibe.Evaluation & InteractionUS-05: As a user, I want the results displayed in a highly readable, responsive table that sorts automatically by distance.US-06: As a user, I want to click a row to open a "Detail Modal" containing a map snippet, a description of the venue, and a direct "Call" or "Visit Website" button.US-07: As a user, I want to click a "Share" icon that copies a unique URL (e.g., kidspot.london/venue/123) to my clipboard.System & AdministrationUS-08: As the platform admin, I want unstructured HTML from church halls to be automatically parsed by an LLM into structured JSON so I don't have to write manual regex scrapers.US-09: As the system, I want to cache frequent geographic searches in Redis for 1 hour to minimize database load and API costs.6. Feature Specifications (V1 Launch)The Smart Search Bar: Powered by Photon/Nominatim. Includes auto-suggest for London boroughs and postcodes.Dynamic Map View: Toggle between "List View" and "Map View" using MapLibre GL JS. Pins cluster when zoomed out and expand when zoomed in.The "Never Zero" Engine: If venues.length === 0, trigger the search fallback API. Inject the results into the UI with a subtle "External Web Results" badge.SEO-Optimized Venue Pages (Programmatic SEO): Every venue gets a statically generated (or SSR) page at /venue/[slug] to capture long-tail search traffic like "Soft play near Hackney".7.
